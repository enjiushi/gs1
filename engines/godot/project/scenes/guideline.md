# engines/godot/project/scenes/

Godot-authored scene assets for the first-party GS1 client shell.

## Usage
- Read this file before scanning the folder in detail.
- When files or folders in this directory change, update this file in the same change.

## Contents
- `main.tscn`: Bootstrap Godot client-shell scene whose root is the native `Gs1GodotDirectorControl`, with a dedicated `SceneHost` child that receives the app-state-selected screen scene while the director-owned adapter service runs the gameplay session and message dispatch.
- `loading.tscn`: Lightweight loading-screen scene used as a transient full-screen shell during async main-menu-to-regional-map scene loading, now including a native-driven progress bar and status label while still keeping the real target scene fully intact so its controllers initialize only after the complete `PackedScene` and queued preload resources are ready to instantiate/use.
- `main_menu.tscn`: Main-menu scene rooted at `Gs1GodotMainMenuUiController`, using the desert shell theme with a warmer left-side atmospheric wash, a slimmer left-anchored menu stack, stronger screen-space heat haze, and a dedicated `SubViewport`-backed authored 3D backdrop scene instead of the old native geometry build-up, while its fixed authored `Start New Campaign`, `Settings`, and `Quit` buttons carry `gs1_panel_id` plus `gs1_slot_id` metadata and the shared `gs1_slot_button` group so the scene-owned native action-panel controller can bind gameplay-projected actions by panel-local slot rather than by projecting concrete buttons, and the button stack now sits inside a plain `MarginContainer` spacer instead of the old translucent `PanelContainer` shell so the three buttons no longer float on a large dark backdrop.
- `animated_menu_killer.tscn`: Menu-only wrapper scene that pairs a hidden `basic_animation_free` driver skeleton with the visible PSX killer mesh so the backdrop character can play a retargeted imported clip without baking project-local gameplay logic into the broader shell.
- `main_menu_backdrop.tscn`: Authored 3D main-menu backdrop scene whose root is `RegionalMapWorld`, still built around the menu-only `Terrain3D` desert terrain plus rock, shelter, water-tower, cactus, and desert-rock dressing to frame a low-altitude right-side camp shot against distant dunes and blue sky, now with editor texture freeing disabled so the router-loaded menu terrain keeps its compiled textures instead of falling back to checker squares a few seconds after startup, and with the duplicated inline terrain normal-map image blobs removed so the scene no longer embeds large serialized binary texture payloads.
- `regional_map.tscn`: Regional-map scene rooted at `Gs1GodotRegionalMapSceneController`, instancing the authored regional desert backdrop and dedicated `regional_map_hud.tscn`, adding the same heat-haze pass used by the menu, keeping the site list and selection shell in the lighter open layout instead of the old full translucent panel shell while still separating site-brief, support-feed, and provided-loadout sections, now wrapping the site brief and the support/loadout subsections in warmer translucent authored panel shells so their text remains readable over the world backdrop, with the provided-loadout subsection still using a dedicated read-only inventory-style slot grid plus empty-state label instead of a single rich-text summary, with the authored duplicate deploy/clear buttons removed so the native selection controller owns the single deploy affordance, and instancing the shared `tech_tree_overlay.tscn` scene so the regional research UI comes from the same reusable unlock-and-tech overlay used by other screens while still binding its authored fixed controls through `gs1_panel_id` plus `gs1_slot_id` metadata, supplying host-specific overlay margins and panel size through `gs1_*` metadata on the overlay instance, and letting the native scene controller own the authored world/camera/picking while sibling scene-owned controller roots own the HUD, selection, summary, and tech overlay runtime subscriptions.
- `regional_map_hud.tscn`: Reusable regional-map HUD panel scene with campaign summary text, selected-site summary text, and the `Research & Unlocks` button owned by the native regional-map HUD controller, now presented without the old `PanelContainer` backdrop so future regional-map buttons can join the same lighter entrance surface.
- `regional_map_backdrop.tscn`: Authored regional-map 3D backdrop scene whose root is `RegionalMapWorld`, combining a broad dune basin, surrounding rock shelves, sparse desert scatter, HDRI lighting, a shared desert `Terrain3D` foundation matching the menu terrain language, and empty runtime-owned `TerrainRoot`/`LinkRoot`/`SiteRoot` anchors so the live campaign markers sit on real terrain instead of generated box tiles.
- `regional_site_marker.tscn`: Authored site-marker scene for the runtime-instanced regional map outposts, built from the desert shelter, water tower, and rock props plus small state-accent meshes, a dedicated selection-ring mesh, and a label anchor so live sites read as camps instead of primitive cylinders while still exposing a clear selected-state cue.
- `site_session.tscn`: Site-session scene rooted at `Gs1GodotSiteSceneController`, which owns the site-world rendering/presenter path directly, plus the instanced `site_hud.tscn`, authored phone and split inventory sub-scenes, task/craft/overlay surfaces, projected protection-selector and site-result modal containers, and the static local controls that still forward host events alongside projected site buttons, now using the shared field-command project theme, the same instanced `tech_tree_overlay.tscn` scene used on the regional map so unlock-and-tech browsing is reusable wherever gameplay opens it, and dedicated task/modifier row containers that the native adapter fills from resource-backed task-template and modifier presentation lookups, with its own host-authored overlay margins and minimum panel size supplied through overlay-instance metadata, while sibling scene-owned controller roots own panel runtime subscriptions and the child `Gs1GodotSiteSessionUiController` keeps only selected-tile and static site command wiring.
- `site_hud.tscn`: Reusable site-session HUD scene instanced by the site shell, now wrapping the authored `player_meter_panel.tscn` sub-scene plus compact launcher buttons for phone, pack, tasks, craft, protection overlays, and research/unlocks so the native HUD controller can open projected panels without rebuilding the broader site layout.
- `player_meter_panel.tscn`: Authored player-condition sub-scene used by the site HUD for projected cash, completion, and the five core worker meter readouts that the native site HUD controller updates from HUD-state messages.
- `phone_panel.tscn`: Authored site-session phone surface used by the sibling native phone-panel controller for tab buttons, section summary, and dynamic listing reconciliation.
- `pack_inventory_panel.tscn`: Authored worker-pack inventory sub-scene used by the site inventory controller for the carried-item slot grid.
- `storage_inventory_panel.tscn`: Authored opened-storage inventory sub-scene used by the site inventory controller for nearby device storage slot grids, remaining hidden until gameplay opens a storage view.
- `site_placeholders/`: Script-free `Node3D` placeholder plant and device scenes keyed from shared gameplay content IDs through the Godot content-resource config, so the site-session adapter can instance distinct temporary art per plant/device type before final assets arrive.
- `tech_tree_overlay.tscn`: Shared unlock-and-tech overlay scene with the centered panel shell and one scrollable reputation ladder that keeps the total-reputation unlockable lane beside the Village, Bureau, and University tech lanes, now using a near-opaque authored panel shell for readability over bright backdrops, with fixed column headers, larger icon-forward cards, and narrow connector-arrow spacer rows between reputation levels so both regional-map and site-session hosts render the same combined progression view.
