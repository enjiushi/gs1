# engines/godot/project/scenes/

Godot-authored scene assets for the first-party GS1 client shell.

## Usage
- Read this file before scanning the folder in detail.
- When files or folders in this directory change, update this file in the same change.

## Contents
- `main.tscn`: Bootstrap Godot client-shell scene whose root is the native `Gs1GodotSceneRouterControl`, keeping the shared `Gs1RuntimeNode` alive while instancing the app-state-selected screen scene into a host container.
- `main_menu.tscn`: Main-menu scene rooted at `Gs1GodotMainScreenControl`, now using the desert shell theme with a warmer left-side atmospheric wash, a slimmer left-anchored menu stack, stronger screen-space heat haze, and a dedicated `SubViewport`-backed authored 3D backdrop scene instead of the old native geometry build-up, while its fixed authored `Start New Campaign`, `Settings`, and `Quit` buttons carry `gs1_panel_id` plus `gs1_slot_id` metadata and the shared `gs1_slot_button` group so the native adapter can bind gameplay-projected actions by panel-local slot rather than by projecting concrete buttons.
- `animated_menu_killer.tscn`: Menu-only wrapper scene that pairs a hidden `basic_animation_free` driver skeleton with the visible PSX killer mesh so the backdrop character can play a retargeted imported clip without baking project-local gameplay logic into the broader shell.
- `main_menu_backdrop.tscn`: Authored 3D main-menu backdrop scene whose root is `RegionalMapWorld`, still built around the menu-only `Terrain3D` desert terrain plus rock, shelter, water-tower, cactus, and desert-rock dressing to frame a low-altitude right-side camp shot against distant dunes and blue sky, now with editor texture freeing disabled so the router-loaded menu terrain keeps its compiled textures instead of falling back to checker squares a few seconds after startup, and with the duplicated inline terrain normal-map image blobs removed so the scene no longer embeds large serialized binary texture payloads.
- `regional_map.tscn`: Regional-map scene rooted at `Gs1GodotMainScreenControl`, now instancing the authored regional desert backdrop and dedicated `regional_map_hud.tscn`, adding the same heat-haze pass used by the menu, keeping the site list and selection shell in the unified translucent desert style with separate site-brief, support-feed, and provided-loadout sections, and instancing the shared `tech_tree_overlay.tscn` scene so the regional research UI comes from the same reusable unlock-and-tech overlay used by other screens while still binding its authored fixed controls through `gs1_panel_id` plus `gs1_slot_id` metadata, supplying host-specific overlay margins and panel size through `gs1_*` metadata on the overlay instance, and letting the native adapter procedurally build the site list from gameplay-projected panel row data.
- `regional_map_hud.tscn`: Reusable regional-map HUD panel scene with campaign summary text, selected-site summary text, and the `Research & Unlocks` button owned by the native regional-map HUD controller so future regional-map buttons can join the same entrance surface.
- `regional_map_backdrop.tscn`: Authored regional-map 3D backdrop scene whose root is `RegionalMapWorld`, combining a broad dune basin, surrounding rock shelves, sparse desert scatter, HDRI lighting, a shared desert `Terrain3D` foundation matching the menu terrain language, and empty runtime-owned `TerrainRoot`/`LinkRoot`/`SiteRoot` anchors so the live campaign markers sit on real terrain instead of generated box tiles.
- `regional_site_marker.tscn`: Authored site-marker scene for the runtime-instanced regional map outposts, built from the desert shelter, water tower, and rock props plus small state-accent meshes and a label anchor so live sites read as camps instead of primitive cylinders.
- `site_session.tscn`: Site-session scene rooted at `Gs1GodotMainScreenControl`, including `Gs1SiteViewNode`, the site HUD panels, inventory/task/phone/craft/overlay surfaces, and the static local controls that still forward host events alongside projected site buttons, now using the shared field-command project theme, shorter in-fiction button labels, a dedicated `Research & Unlocks` control in the site command stack, the same instanced `tech_tree_overlay.tscn` scene used on the regional map so unlock-and-tech browsing is reusable wherever gameplay opens it, and dedicated task/modifier row containers that the native adapter fills from resource-backed task-template and modifier presentation lookups, with its own host-authored overlay margins and minimum panel size supplied through overlay-instance metadata.
- `tech_tree_overlay.tscn`: Shared unlock-and-tech overlay scene with the centered panel shell and one scrollable reputation ladder that keeps the total-reputation unlockable lane beside the Village, Bureau, and University tech lanes, using fixed column headers, larger icon-forward cards, and narrow connector-arrow spacer rows between reputation levels so both regional-map and site-session hosts render the same combined progression view.
