# engines/godot/native/

Native C++ source for the Godot runtime extension target, now owned by a `godot-cpp`-first adapter that talks to the engine through normal `godot-cpp` classes and APIs while keeping the gameplay DLL boundary in the shared host/session layer, with controller setup expected to happen through lifecycle entry points such as `_ready()` instead of being retried from per-frame ticks.

## Usage
- Read this file before scanning the folder in detail.
- When files or folders in this directory change, update this file in the same change.

## Contents
- `gs1_godot_extension_entry.cpp`: `godot-cpp` extension entry point that registers `Gs1GodotDirectorControl` plus the split scene and UI controller roots for the main-menu/regional-map/site-session shells through the normal binding bootstrap.
- `gs1_godot_adapter_service.h` / `gs1_godot_adapter_service.cpp`: Pure C++ adapter service that owns the gameplay runtime session, loads the staged runtime/config roots, drains engine messages directly from the DLL queue, dispatches them through per-message subscriber buckets, and forwards host-event submissions back into gameplay.
- `gs1_godot_controller_context.h` / `gs1_godot_controller_context.cpp`: Small controller lookup helpers that let scene/UI/presenter nodes find the active `Gs1GodotDirectorControl` and its shared adapter service without hard-wiring controller-to-controller references.
- `gs1_godot_projection_types.h`: Shared Godot-adapter projection structs and aliases used by the adapter service, reducers, scene controllers, and panel controllers without forcing them to include one centralized main-screen cache header.
- `gs1_godot_panel_state_reducers.h` / `gs1_godot_panel_state_reducers.cpp`: Message-driven reducer helpers that now split gameplay projection assembly by responsibility such as status, UI setups, UI panels, progression views, regional map, and site session state, so live message-derived state can move into the specific controller that owns each UI surface instead of one global main-screen cache.
- `gs1_godot_status_panel_controller.h` / `gs1_godot_status_panel_controller.cpp`: Scene-owned panel controller for the status feed label, its reducer-backed app/campaign/action summary state, runtime-link state, and self-refresh logic without screen-shell orchestration.
- `gs1_godot_overlay_panel_controller.h` / `gs1_godot_overlay_panel_controller.cpp`: Small direct-subscriber UI controller for the overlay panel, owning the currently presented overlay mode, projected protection-selector and site-result modal setup reconciliation, site-state panel visibility, the local overlay-mode button wiring, and direct label refresh straight from the message stream without routing through a shared main-screen cache.
- `gs1_godot_regional_summary_panel_controller.h` / `gs1_godot_regional_summary_panel_controller.cpp`: Direct-subscriber controller for the regional map summary/feed labels, owning only the campaign-resource plus regional-map presentation state it needs and no longer reading another controller's data.
- `gs1_godot_site_summary_panel_controller.h` / `gs1_godot_site_summary_panel_controller.cpp`: Direct-subscriber controller for the site summary/title labels, owning only the site summary presentation state it needs from site, HUD, and action messages instead of borrowing data from the status controller.
- `gs1_godot_site_hud_controller.h` / `gs1_godot_site_hud_controller.cpp`: Scene-owned direct-subscriber controller for the site-session HUD scene, owning projected player-meter/cash/completion labels, top-level site panel launcher buttons, and small badge state while submitting HUD intents through the adapter service.
- `gs1_godot_inventory_panel_controller.h` / `gs1_godot_inventory_panel_controller.cpp`: Direct-subscriber controller for the inventory panel, locally assembling only the worker-pack and opened-storage presentation state required for that panel and now also owning that panel's site-state visibility.
- `gs1_godot_phone_panel_controller.h` / `gs1_godot_phone_panel_controller.cpp`: Scene-owned direct-subscriber controller for the phone panel, owning the section summary label, site-state panel visibility, the phone-tab plus close-button input wiring, dynamic phone-listing button reconciliation, and phone intent submission.
- `gs1_godot_craft_panel_controller.h` / `gs1_godot_craft_panel_controller.cpp`: Scene-owned direct-subscriber controller for the craft/placement panel, owning the craft/placement summary label, site-state panel visibility, dynamic craft-option button reconciliation, and craft-request submission.
- `gs1_godot_task_panel_controller.h` / `gs1_godot_task_panel_controller.cpp`: Direct-subscriber controller for the site task/modifier panel, owning site-state panel visibility, the task-summary label, and the disabled task/modifier row reconciliation plus its local icon/metadata cache instead of leaving that panel in the main screen shell.
- `gs1_godot_action_panel_controller.h` / `gs1_godot_action_panel_controller.cpp`: Scene-owned direct-subscriber controller for fixed-slot and projected site-action wiring, owning the UI-setup and UI-panel reducer slices needed to bind slot actions, the static protection-overlay opener, dynamic site control buttons, and UI-action submission.
- `gs1_godot_regional_map_hud_controller.h` / `gs1_godot_regional_map_hud_controller.cpp`: Direct-subscriber controller for the regional-map HUD scene, owning campaign summary text, selected-site hint text, and the tech-panel entry button so future regional-map HUD buttons can be added without routing those panel-local controls through the shell.
- `gs1_godot_regional_selection_panel_controller.h` / `gs1_godot_regional_selection_panel_controller.cpp`: Direct-subscriber controller for the regional selection panel, owning the selected-site brief plus dedicated support-feed and provided-loadout summaries alongside deployment action-button reconciliation while following authoritative regional-map messages for selected-site state instead of accepting shell-pushed selection, now also keeping those panel-local button signals inside the controller instead of routing them back through the shell.
- `gs1_godot_regional_tech_tree_panel_controller.h` / `gs1_godot_regional_tech_tree_panel_controller.cpp`: Scene-owned direct-subscriber controller for the regional research overlay, owning the progression-view reducer, overlay/panel visibility, the unlockable/technology card reconciliation, the close-button plus card-action input wiring, icon/tooltip caches, and tech action-button routing instead of leaving the tech-tree overlay in the main screen shell.
- `gs1_godot_director_control.h` / `gs1_godot_director_control.cpp`: Bootstrap `godot-cpp` `Control` that owns the shared adapter service, watches the authoritative projected app state, and swaps the menu/regional/site scene instance without synthesizing bootstrap gameplay messages.
- `gs1_godot_main_menu_ui_controller.h` / `gs1_godot_main_menu_ui_controller.cpp`: Main-menu UI root that owns the shell-local button wiring and start/quit runtime submissions without carrying any regional-map or site-session responsibilities.
- `gs1_godot_regional_map_scene_controller.h` / `gs1_godot_regional_map_scene_controller.cpp`: Regional-map scene root that owns world-node lookup, regional snapshot reconciliation, authored camp marker visuals, camera framing, and regional click-picking/input dispatch while leaving HUD/panel behavior to dedicated UI controllers.
- `gs1_godot_regional_map_ui_controller.h` / `gs1_godot_regional_map_ui_controller.cpp`: Regional-map UI root that owns app-state plus UI-surface visibility handling for the HUD, selection shell, summary shell, and shared tech-tree overlay without owning or subscribing panel controllers.
- `gs1_godot_site_scene_controller.h` / `gs1_godot_site_scene_controller.cpp`: Site-session scene root and world presenter that owns the bootstrap camera/light, projected tile/plant/device/worker visuals, gameplay-ID-to-Godot-instance reconciliation, and direct runtime-message subscription for site rendering without a separate site-view controller node.
- `gs1_godot_site_session_ui_controller.h` / `gs1_godot_site_session_ui_controller.cpp`: Site-session UI root that owns selected-tile state, static site command wiring, and the small remaining shell-level host submissions without owning or subscribing the scene-owned status, HUD, inventory, phone, craft, task, action, overlay, summary, or tech-tree panel controllers.
- `godot_progression_resources.h` / `godot_progression_resources.cpp`: Lightweight Godot-only TOML loader for progression and broader render resources, keyed by shared `tech_node_id`, unlockable content kind/id pairs, general content kind/id pairs, and domain/id entries such as modifiers, task templates, reward candidates, and sites so the adapter can resolve icon, thumbnail, and scene paths without moving shared descriptions or gameplay meaning out of the gameplay-authored content tables, now loading from the staged `res://gs1/godot_config/` package copy.
