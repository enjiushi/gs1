# src/content/tables/

External prototype content TOML files loaded by the gameplay DLL at startup.

## Usage
- Read this file before scanning the folder in detail.
- When files or folders in this directory change, update this file in the same change.

## Contents
- `campaign_setup.toml`: Prototype campaign setup content for per-site-session starting cash, support quota per contributor, and baseline deployment items, with authored cash normalized to internal cash points at load.
- `sites.toml`: Prototype campaign/site authoring content for regional-map tile placement, objective config, support exports, nearby aura seeds, completion rewards, camp anchors, site default-weather baselines including Site 1's hotter starting heat, authored starter-plant patches such as Site 1's harvestable near-camp Desert Ephedra growth, initial reveal/availability flags, and per-site phone delivery fees authored in player cash but normalized to internal cash points at load.
- `phone_listings.toml`: Seeded phone-listing content for non-refreshing per-site offers such as contractor slots and any future direct-purchase unlockables, with buy-item stock now authored separately through the generated stock-pool table.
- `phone_buy_stock.toml`: Per-site phone buy-stock pool rows keyed by stable listing ids, each authoring a survival/basic-material stock item or plant-seed item with a target internal-cash-point budget plus variance so runtime refreshes can reroll limited quantities, while seed entries appear only when their linked plants are unlocked.
- `items.toml`: Item authoring content for runtime item definitions and item-domain metadata, including consumable meter deltas that can derive consumable internal cash-point valuation from the shared player-meter cash-point tuning, optional shared `modifier_id` references that add the current flat timed-modifier premium when the referenced modifier duration is nonzero, fallback internal cash-point valuation for non-meter goods used by task/reward scoring and phone/store pricing, the full prototype seed-item roster, unique harvested-good items for every harvestable desert plant, the shovel tool, the baseline Ephedra Stew onboarding food plus the later Village cooked-food outputs, and the current excavation-only stone merchandise balance rows.
- `excavation_tables.toml`: Excavation depth tuning and depth-scoped tier `100%` loot pools for the prototype desert stone finds, including depth-specific rough/careful/thorough dig durations, the current `50%` any-find balance, and distinct within-tier splits per depth.
- `plants.toml`: Plant authoring content for ecology, explicit focus tags, the authored six-meter roster-pool model (`salt`, `heat`, `wind`, `dust`, `fertility`, `output`), footprint-multiple aura and wind shelter reach, authored nearby-protection ratios over resistance-driven outward shelter, salt-derived plant salinity rehab for support foci, the straw checkerboard's steady five-real-minute constant wither behavior, and per-plant harvest thresholds/output tuning across the full prototype China-desert 10-plant roster, now with one authored primary harvest item per plant plus secondary dried-grass output.
- `structures.toml`: Structure authoring content for buildables, storage, crafting stations, and authored resolved-tile support channels such as wind, heat, dust, fertility, salinity, and irrigation output.
- `craft_recipes.toml`: Craft recipe authoring content for station/output/ingredient mappings plus recipe-authored craft duration and worker energy/hydration/nourishment costs, including the baseline Ephedra Stew onboarding recipe, the shovel recipe, the Village broth/tonic/hotpot/refief-food chains, and the older buff-granting cooked drink and meal examples.
- `task_templates.toml`: Authored task template content using single target/required-count/threshold values, where `0` or `None` means the runtime must resolve that field procedurally, plus authored expected in-game hours and risk multipliers for valuation and optional selector hints for runtime-picked item/plant/recipe/structure/action values.
- `site_onboarding_task_seeds.toml`: Site-keyed onboarding task seed content that instantiates exact task instances for a specific site, including exact follow-up task chaining plus up to four fixed bundled reward-candidate references for the Site 1 phone tutorial flow.
- `reward_candidates.toml`: Reward candidate content for money, deliveries, run modifiers, and reputation rewards, with authored cash grants normalized to internal cash points at load and the Site 1 onboarding flow now bundling several single-effect candidates together through the onboarding seed table.
- `site_actions.toml`: Site action tuning content for shared action duration and worker energy/hydration/nourishment/morale costs, authored local-weather-to-meter cost coefficients, placement occupancy, and reservation/approach/movement flags, with craft duration and worker meter costs authored per recipe, the shared harvest action profile, and the authored excavate action profile loaded here.
- `modifier_presets.toml`: Unified modifier content for nearby auras, permanent run modifiers, and timed consumable modifiers, with every entry authored behind an explicit `modifier_id`, optional display text, `0` duration for permanent effects, positive `8`-hour duration blocks for timed effects, optional `timed_buff_cap_bias`, worker action-cost modifier deltas, Village custom food buffs, and generated linear-tech modifier rows such as the split Village timed-buff-cap expansions.
- `gameplay_tuning.toml`: Shared numeric gameplay-tuning table for worker-condition, player-meter internal cash-point valuation, item buy/sell multipliers, ecology, modifier, device-support, camp-durability, and periodic task/phone refresh cadence factors, including worker background energy-recovery tuning, the modifier system's configurable base active timed-buff cap, the current higher base plant-pressure floor with lighter dust weighting, the shared `0.01` weather-vs-resistance growth-pressure comparison that should remain data-authored instead of hardcoded, and the ecology full-range real-minutes tuning for the linear density-change curve, currently tuned to a `5` real-minute full-range baseline.
- `technology_tiers.toml`: Faction tech-tree tier content for tier ordering and display names across the full authored `32`-tier branch.
- `reputation_unlocks.toml`: Total-reputation unlock content for the authored access ladder spanning plants, items, recipes, and the currently craft-backed structure recipes, now covering only the non-starter progression content and stepping by `200` reputation from the first unlock at `200`.
- `technology_nodes.toml`: Technology node content for one linear faction-owned tech per tier, authored with `200 * tier` faction-reputation requirements from `200` through `6400`, zero current claim-cost cash-point values because tech rows auto-unlock from faction reputation, entry kinds, and the current Village/Bureau/University split across incremental Village mechanics plus stretched Bureau/University branch effects.
- `initial_unlocked_plants.toml`: Starter-plant content for the two plants available from campaign start before later reputation unlocks.
