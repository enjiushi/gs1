# src/content/defs/

Definition types and runtime accessors for the major authored content domains.

## Usage
- Read this file before scanning the folder in detail.
- If you open a child folder, read its `guideline.md` first when present.
- When files or folders in this directory change, update this file in the same change.

## Contents
- `campaign_content_defs.h`: Campaign-level authored content records.
- `faction_defs.h`: Prototype faction identity and assistant unlock definition records.
- `craft_recipe_defs.h`: Crafting recipe definition types plus runtime accessors over the loaded recipe table, including recipe-authored craft duration and worker energy/hydration/nourishment costs plus the Village shovel, Chemistry Station kit, Herbal Poultice, Wind Fence kit, broth, tonic, hotpot, and capstone relief-food recipe ids.
- `excavation_defs.h`: Excavation depth/tier authoring types plus runtime accessors for depth-specific duration, energy/find tuning, and depth-scoped tier loot pools.
- `event_defs.h`: Authored event definitions used by event/state systems.
- `gameplay_tuning_defs.h`: Shared gameplay-tuning definition records and lookup for authored numeric factors consumed by worker-condition, player-meter internal cash-point valuation, plant-harvest dedicated-output budgets plus plant-pool-derived harvest-good and seed pricing ratios, item buy/sell price derivation, ecology, modifier, device-support, and camp-durability systems, including worker background energy-recovery tuning, the simplified weather-vs-resistance growth-pressure scaling, the modifier system's configurable base active timed-buff cap, and the authored full-range real-minutes control for the linear density-change curve, with the fallback baseline set to `5` real minutes.
- `item_defs.h`: Item definition types plus runtime accessors over the loaded item table, including fallback internal cash-point values for non-meter goods plus player-meter-derived valuation for consumables, shared `modifier_id` references that can point at permanent or timed run modifiers and still add the current flat timed-modifier premium when duration is nonzero, derived buy/sell internal pricing helpers backed by shared multipliers instead of authored item cash fields, plant-linked derived harvest-good and seed pricing from total plant meter pools, the starter checkerboard item, the full prototype 10-plant seed roster, per-plant harvest-good ids including the new Village cooking ingredients, the shovel, Wind Fence device-kit, Chemistry Station device-kit, and Herbal Poultice medicine-item ids, Village cooked-food item ids, and excavation-only stone merchandise ids.
- `modifier_defs.h`: Unified modifier definition records keyed by explicit modifier ids, with nearby-aura versus run-modifier kind tags, optional authored display name/description, duration-driven permanent-versus-timed behavior, timed-buff-cap-bias payloads, action-cost modifier deltas, harvest-output cash-point multiplier deltas, and runtime accessors applied by modifier/weather flows.
- `plant_defs.h`: Plant/ecology definition types plus runtime accessors over the loaded plant table, including the full China-desert 10-plant prototype roster, explicit plant focus tags, the authored six-meter roster-pool model (`salt`, `heat`, `wind`, `dust`, `fertility`, `output`), derived salt-to-salinity support helpers for support-focused plants, authored footprint-multiple shelter reach plus nearby-protection ratios, table-driven per-plant harvest-output metadata with guaranteed/fixed-chance/budget-scaled rows, and the starter straw checkerboard definition with its authored `2x2` footprint plus tuned steady-wither behavior used by directional shelter resolution.
- `recipe_defs.h`: General recipe definition records shared by authored content.
- `reward_defs.h`: Reward definition records and runtime accessors for grants and payouts, including the shared prototype task-reward pool with internal-cash-point money awards, simple modifier rewards, immediate unlockable reveals, and delivery-crate-routed item bundle candidates.
- `site_content_defs.h`: Site template/content definitions for active site runs.
- `structure_defs.h`: Structure/buildable definition types plus runtime accessors over the loaded structure table, including authored multi-channel support values for resolved tile contribution state and the dedicated Chemistry crafting-station kind used by medicine recipes.
- `task_defs.h`: Task board/authored task records plus runtime accessors, including single-value target/required-count/threshold fields that use `0` or `None` as procedural sentinels, authored expected in-game hours and risk multipliers for task valuation, expanded progress kinds such as living-plant non-withering duration checks plus item submission, optional action/item/plant/recipe/structure selector hints, and site-keyed onboarding task seed overrides with optional fixed reward candidates plus the dedicated Site 1 hammer-before-device/tool crafting onboarding step id.
- `technology_defs.h`: Prototype progression definition records and runtime accessors, including the linear `32`-tier-per-faction tech ladder, currently zeroed per-node cash-point claim costs because tech rows auto-unlock from faction reputation, authored reputation-scaled effect parameters, the shared total-reputation unlock ladder across plants/items/recipes/device-build recipes with starter Field Kitchen, Workbench, Storage Crate, Hammer, Wood, Iron, and base Medicine plus early Chemistry Station and Herbal Poultice progression and the later Wind Fence unlock, starter plants available from campaign start, and placeholder-friendly faction tech nodes without enhancement-choice metadata.
