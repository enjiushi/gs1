# src/runtime/

Core runtime services that drive fixed-step execution, gameplay-message routing, clocks, and host integration.

## Usage
- Read this file before scanning the folder in detail.
- If you open a child folder, read its `guideline.md` first when present.
- When files or folders in this directory change, update this file in the same change.

## Contents
- `engine_message_queue.h`: Queue helpers for engine-originated messages entering gameplay.
- `fixed_step_runner.h`: Fixed-timestep runner used by the runtime update loop.
- `game_systems.h`: Canonical compile-time ordered gameplay system-pack declaration mirroring the active runtime registry order for the typed internal-dispatch refactor.
- `gameplay_message_traits.h`: Transitional typed-gameplay-message-to-legacy-envelope mapping helpers used while migrating internal gameplay dispatch away from enum-plus-payload `GameMessage` transport, now also covering the typed site-start, placement-reservation, worker-meter, tile-state, and task-reward follow-up message families while non-migrated gameplay traffic still bridges through the legacy envelope path.
- `game_runtime.h`: Main runtime type declarations and split update entry points, with `GameRuntime` owning the `StateManager`, state-view cache, host-message queue, runtime-message queue access, and profiling toggles/snapshots, now carrying an O(1) system-pointer table keyed by compile-time `GameSystems` pack index so typed internal dispatch no longer linearly scans systems with `dynamic_cast`, with typed inline dispatch now covering site-start plus additional site-runtime follow-up families such as worker-meter, tile-state, several reward/progression messages, and site-scene activation, now also keeping a debug-only semantic gameplay-message stack for nested inline dispatch visibility, now also caching per-subscriber profile metadata for the legacy `GameMessage` dispatch path during migration, and with the runtime no longer round-tripping through production aggregate compatibility mirrors before or after dispatch.
- `game_state.h`: Compatibility state access alias for the runtime-owned authoritative state-set storage now held by `StateManager`, with host and runtime message transports still stored on the runtime-owned aggregate while the current architecture direction moves internal gameplay-message handling away from deferred queue drains and toward inline nested dispatch.
- `game_runtime.cpp`: Runtime loop implementation and routing core, including direct system creation, resolver registration, split host-message phase dispatch, internal gameplay message dispatch, profiling, fixed-step scheduling, rebuilding/querying the public gameplay-state view from authoritative split state storage, privileged lifecycle-only campaign/site snapshot install helpers so flow systems do not bypass owner checks, the stack-based mutating-system restoration needed for re-entrant nested gameplay-message handling, the typed inline dispatch scaffolding for migrated gameplay-message families, the shared debug-only semantic nested-message stack capture for both typed and legacy inline dispatch, the compile-time-pack-order system-pointer table fill used by O(1) typed system lookup during migration, compile-time-trait-backed fixed-step/profile metadata caching for the full active `GameSystems` pack plus cached legacy-message subscriber profiling metadata, direct runtime-log transport helpers used by migrated internal debug probes, and the now-thinner legacy envelope/queue dispatch core that first bridges already-migrated gameplay-message families back into compile-time typed subscriber dispatch while leaving only the unmigrated families on the enum-keyed subscriber table.
- `game_state_view.h` / `game_state_view.cpp`: Runtime-owned helpers that materialize the exported ABI-safe `Gs1GameStateView`, rebuild host-facing cache slices directly from flat authoritative state sets, flatten active-site craft-context and placement-preview data for host reads, derive the worker sheltered flag from live site weather instead of storing it on worker vitals, and answer targeted tile queries without exposing internal runtime storage directly.
- `runtime_state_access.h`: Tagged runtime-state access declarations plus `RuntimeInvocation` plumbing for authoritative app/campaign/site state-set access, fixed-step timing, message transport access, and move-direction snapshots, now with explicit tagged owner mapping plus const-only non-owner reads so `GameStateAccess::write()` stays owner-only, including the dedicated campaign-progression tag used by the generic reputation/token authority.
- `system_pack.h`: Small compile-time ordered system-pack helper for the typed internal-dispatch refactor.
- `runtime_split_state_compat.h`: Shared split-state helper mappings for fine-grained state slices plus shared aggregate campaign/site snapshot writers and readers used by runtime lifecycle install/reset paths and the temporary aggregate compatibility cache, while the remaining test-only aggregate round-trip helpers now thinly wrap these shared runtime helpers under `tests/split_state_test_helpers.h`.
- `state_manager.h` / `state_manager.cpp`: Split-state resolver registration plus typed authoritative access for the authoritative app, campaign, and site state sets, with non-container wrappers cache-line aligned at 64 bytes, the manager now owning the aggregated `GameState`, and runtime mutation guards that reject cross-owner state writes outside privileged lifecycle installation while using a mutating-system stack so nested inline gameplay-message dispatch restores outer owner context correctly.
- `state_set.h`: State-set id inventory, cache-line-aligned wrapper templates, and compile-time 64-byte alignment enforcement for non-container state sets used by the state-resolver refactor, plus the separate non-aligned container wrapper for container-backed state sets and the pointer-free payload rule for both non-container states and container elements, now also including the dedicated `CampaignProgression` split state for generic token balances and fired-threshold memory.
- `system_interface.h`: Shared runtime system interface and subscriber array declarations, now also carrying state-set ownership helpers that map explicit runtime claims and site-component ownership to `StateSetId` registration plus compile-time trait readers for typed subscribed-message, emitted-runtime-message, fixed-step-order, and profile-id metadata with runtime virtual fallbacks during migration, including aggregation of the active `GameSystems` pack's first compile-time runtime-to-host message manifest.
- `type_list.h`: Shared compile-time type-list helpers used by runtime state tags, system-pack declarations, and the typed internal gameplay-dispatch refactor, now also covering unique-union and size helpers used by the runtime emitted-message manifest scaffold.
- `id_allocator.h`: Lightweight runtime ID allocation helpers.
- `message_queue.h`: Legacy/traditional gameplay message queue declarations that still back the remaining unmigrated internal gameplay-message families while the architecture direction moves toward immediate inline typed internal gameplay-message dispatch only.
- `random_service.h`: Runtime random-number service/state wrapper.
- `runtime_clock.h`: Canonical runtime day-length constants plus real-seconds-to-in-game-minutes conversion helpers.
