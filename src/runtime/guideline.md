# src/runtime/

Core runtime services that drive fixed-step execution, gameplay-message routing, clocks, and host integration.

## Usage
- Read this file before scanning the folder in detail.
- If you open a child folder, read its `guideline.md` first when present.
- When files or folders in this directory change, update this file in the same change.

## Contents
- `engine_message_queue.h`: Queue helpers for engine-originated messages entering gameplay.
- `fixed_step_runner.h`: Fixed-timestep runner used by the runtime update loop.
- `game_systems.h`: Canonical compile-time ordered gameplay system-pack declaration mirroring the active runtime registry order for the typed internal-dispatch refactor.
- `gameplay_message_traits.h`: Transitional typed-gameplay-message-to-legacy-envelope mapping helpers used while migrating internal gameplay dispatch away from enum-plus-payload `GameMessage` transport, now also covering the typed site-start, placement-reservation, worker-meter, tile-state, task-reward follow-up, task accept/claim request, modifier-end request, storefront buy/sell/hire/unlock requests, and technology-claim/refund message families while non-migrated gameplay traffic still bridges through the legacy envelope path.
- `game_runtime.h`: Main runtime type declarations and split update entry points, with `GameRuntime` owning the `StateManager`, state-view cache, typed pending host-command queue, runtime-message queue access, profiling toggles/snapshots, and now the active gameplay systems directly in compile-time `GameSystems::tuple_type` storage, with typed inline dispatch now covering site-start plus additional site-runtime follow-up families such as worker-meter, tile-state, several reward/progression messages, and site-scene activation, now also keeping a debug-only semantic gameplay-message stack for nested inline dispatch visibility, and with the runtime no longer round-tripping through production aggregate compatibility mirrors before or after dispatch or maintaining a runtime-built enum-keyed internal gameplay subscriber table. Fixed-step registration is now derived directly from the compile-time pack order rather than a runtime sort pass.
- `game_state.h`: Compatibility state access alias for the runtime-owned authoritative state-set storage now held by `StateManager`, with host and runtime message transports still stored on the runtime-owned aggregate while the current architecture direction keeps internal gameplay-message handling on immediate inline dispatch and leaves the legacy internal queue only for non-runtime compatibility fixtures.
- `game_runtime.cpp`: Runtime loop implementation and routing core, including compile-time ordered tuple-backed system registration, resolver registration, split host-command phase dispatch, internal gameplay message dispatch, profiling, fixed-step scheduling, rebuilding/querying the public gameplay-state view from authoritative split state storage, privileged lifecycle-only campaign/site snapshot install helpers so flow systems do not bypass owner checks, the stack-based mutating-system restoration needed for re-entrant nested gameplay-message handling, the typed inline dispatch scaffolding for migrated gameplay-message families, the shared debug-only semantic nested-message stack capture for both typed and legacy inline dispatch, compile-time-trait-backed fixed-step/profile metadata caching for the full active `GameSystems` pack, direct runtime-log transport helpers used by migrated internal debug probes, and the runtime-side translation path from typed public host commands into internal gameplay messages for campaign-flow, technology, task-board, site-modifier, economy-phone, site-inventory-slot, site-context/action, and site-move-direction families. Fixed-step registration now walks the compile-time system pack order directly, and the phase loop no longer performs a runtime queue drain for gameplay coordination.
- `game_state_view.h` / `game_state_view.cpp`: Runtime-owned helpers that materialize the exported ABI-safe `Gs1GameStateView`, rebuild host-facing cache slices directly from flat authoritative state sets, flatten active-site craft-context and placement-preview data for host reads, derive the worker sheltered flag from live site weather instead of storing it on worker vitals, and answer targeted tile queries without exposing internal runtime storage directly.
- `runtime_state_access.h`: Tagged runtime-state access declarations plus `RuntimeInvocation` plumbing for authoritative app/campaign/site state-set access, fixed-step timing, message transport access, and move-direction snapshots, now with explicit tagged owner mapping plus const-only non-owner reads so `GameStateAccess::write()` stays owner-only, including the dedicated campaign-progression tag used by the generic reputation/token authority.
- `system_pack.h`: Small compile-time ordered system-pack helper for the typed internal-dispatch refactor, now with corrected index resolution for the runtime's O(1) typed system lookup path.
- `runtime_split_state_compat.h`: Shared split-state helper mappings for fine-grained state slices plus shared aggregate campaign/site snapshot writers and readers used by runtime lifecycle install/reset paths and the temporary aggregate compatibility cache, while the remaining test-only aggregate round-trip helpers now thinly wrap these shared runtime helpers under `tests/split_state_test_helpers.h`.
- `state_manager.h` / `state_manager.cpp`: Split-state resolver registration plus typed authoritative access for the authoritative app, campaign, and site state sets, with non-container wrappers cache-line aligned at 64 bytes, the manager now owning the aggregated `GameState`, and runtime mutation guards that reject cross-owner state writes outside privileged lifecycle installation while using a mutating-system stack so nested inline gameplay-message dispatch restores outer owner context correctly.
- `state_set.h`: State-set id inventory, cache-line-aligned wrapper templates, and compile-time 64-byte alignment enforcement for non-container state sets used by the state-resolver refactor, plus the separate non-aligned container wrapper for container-backed state sets and the pointer-free payload rule for both non-container states and container elements, now also including the dedicated `CampaignProgression` split state for generic token balances and fired-threshold memory.
- `system_interface.h`: Shared runtime system interface declarations plus state-set ownership helpers that map explicit runtime claims and site-component ownership to `StateSetId` registration, along with compile-time trait readers for typed subscribed-message, emitted-runtime-message, fixed-step-order, and profile-id metadata and aggregation of the active `GameSystems` pack's runtime-to-host message manifest.
- `type_list.h`: Shared compile-time type-list helpers used by runtime state tags, system-pack declarations, and the typed internal gameplay-dispatch refactor, now also covering unique-union and size helpers used by the runtime emitted-message manifest scaffold.
- `id_allocator.h`: Lightweight runtime ID allocation helpers.
- `message_queue.h`: Legacy/traditional gameplay message queue declarations kept only for non-runtime compatibility fixtures while the architecture direction keeps moving toward immediate inline typed internal gameplay-message dispatch only.
- `random_service.h`: Runtime random-number service/state wrapper.
- `runtime_clock.h`: Canonical runtime day-length constants plus real-seconds-to-in-game-minutes conversion helpers.
