# tests/smoke/

DLL-loading smoke harness, scripted smoke runner, live-state server, and visual smoke viewer support.

## Usage
- Read this file before scanning the folder in detail.
- If you open a child folder, read its `guideline.md` first when present.
- When files or folders in this directory change, update this file in the same change.

## Contents
- `assets/`: Static assets used by the visual smoke experience.
- `runtime_dll_loader.h`: Runtime DLL loader interface used by smoke hosts.
- `runtime_dll_loader.cpp`: Runtime DLL loader implementation.
- `scripts/`: Scripted smoke scenario files.
- `smoke_engine_host.h`: Smoke host abstraction over the gameplay DLL runtime, including the thread-safe incoming command queue and published immutable live-state snapshot used by the visual smoke host.
- `smoke_engine_host.cpp`: Smoke host implementation, including frame-thread ownership of runtime API calls and projection mutation, incoming command draining, published snapshot/patch refresh, authoritative site phone home/app-panel state with completed/claimed task counts, site-tile local-wind plus ecology-meter projection intake for viewer-reactive plants and terrain shading, weather-event timeline projection application, campaign cash/total-reputation projection intake, and log-level-aware filtering so debug gameplay logs stay out of activity-only smoke output while remaining available in verbose mode.
- `smoke_host_main.cpp`: Standard smoke host executable entry point, including the optional `--verbose` switch for full engine logging.
- `smoke_live_http_server.h`: HTTP server interface for exposing live smoke state.
- `smoke_live_http_server.cpp`: HTTP server implementation for the visual smoke viewer.
- `smoke_live_state_json.cpp`: JSON serialization of live smoke/runtime state, including tracked worker-pack visibility, phone home/app-panel section/count/open snapshot fields, claimed-task list-kind names, site-tile local-wind plus ecology-meter fields for plant animation and terrain shading, weather-event timeline fields, campaign cash/total-reputation fields, and regional-map tech-tree tab/action/setup names.
- `smoke_log.h`: Smoke logging helper declarations.
- `smoke_log.cpp`: Smoke logging helper implementation.
- `smoke_script_directive.h`: Parsed smoke-script directive/value definitions.
- `smoke_script_runner.h`: Smoke script runner interface.
- `smoke_script_runner.cpp`: Smoke script runner implementation, including tech-tree and phone-panel close UI action parsing for scripts.
- `visual_smoke_host_main.cpp`: Visual smoke host executable entry point, including `--port`, optional `--verbose` handling, queue-driven host command submission from HTTP callbacks, the separate site-control lock, tech-tree plus phone-panel close UI action parsing for browser-driven clicks, and the browser-driven harvest action parser.
- `visual_smoke_live.html`: Browser UI shell and HUD/panel styling for the live visual smoke viewer, including the simplified four-app phone home launcher, the enlarged handset, full-height app panels, inline SVG app and inventory icon surfaces, section scrolling styles, the campaign cash/reputation resource cards used in both site and regional-map views, the tuned site-vitals palette used during site play, the centered site-result modal treatment, and the centered half-screen tech-tree overlay treatment shared by the regional map and active site sessions, now with the two-tab unlockables-versus-tech-tree presentation plus locked-tier dim/lock treatments.
- `visual_smoke_live_viewer.js`: Client-side scene, HUD, inventory, storage, phone panel, centered site-result overlay, and interaction logic for the live visual smoke viewer, including delivery-box
