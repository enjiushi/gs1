# tests/smoke/

DLL-loading smoke harness, scripted smoke runner, live-state server, and visual smoke viewer support.

## Usage
- Read this file before scanning the folder in detail.
- If you open a child folder, read its `guideline.md` first when present.
- When files or folders in this directory change, update this file in the same change.

## Contents
- `assets/`: Static assets used by the visual smoke experience.
- `runtime_dll_loader.h`: Runtime DLL loader interface used by smoke hosts.
- `runtime_dll_loader.cpp`: Runtime DLL loader implementation.
- `scripts/`: Scripted smoke scenario files.
- `smoke_engine_host.h`: Smoke host abstraction over the gameplay DLL runtime, including the thread-safe incoming command queue, published immutable live-state snapshot used by the visual smoke host, recent one-shot cue projection buffering for browser-side audio playback, and site-protection overlay snapshot fields, with phone/hud/campaign money projections stored as player-facing cash floats.
- `smoke_engine_host.cpp`: Smoke host implementation, including frame-thread ownership of runtime API calls and projection mutation, incoming command draining, published snapshot/patch refresh, authoritative site phone home/app-panel state with completed/claimed task counts, site-tile local-wind plus ecology-meter, excavation-depth, and protection-channel projection intake for viewer-reactive plants, terrain shading, excavation marks, and protection heatmaps, host-side conversion of overlay protection channels into final displayed per-tile shelter by combining received protection with density-scaled self heat/wind/dust plant resistance on occupied plant tiles, site-protection overlay state projection application, weather-event timeline projection application, campaign cash/total-reputation projection intake, recent one-shot cue capture for browser audio, and log-level-aware filtering so debug gameplay logs stay out of activity-only smoke output while remaining available in verbose mode.
- `smoke_host_main.cpp`: Standard smoke host executable entry point, including the optional `--verbose` switch for full engine logging.
- `smoke_live_http_server.h`: HTTP server interface for exposing live smoke state.
- `smoke_live_http_server.cpp`: HTTP server implementation for the visual smoke viewer.
- `smoke_live_state_json.cpp`: JSON serialization of live smoke/runtime state, including tracked worker-pack visibility, phone home/app-panel section/count/open snapshot fields, claimed-task list-kind names, site-tile local-wind plus ecology-meter, excavation-depth, and protection fields for plant animation, terrain shading, excavation marks, and protection overlays, site-protection selector/setup and overlay-mode names, weather-event timeline fields, campaign cash/total-reputation fields now serialized as player-facing cash floats, recent one-shot cue payloads for browser audio synthesis, and regional-map tech-tree tab/action/setup names.
- `smoke_log.h`: Smoke logging helper declarations.
- `smoke_log.cpp`: Smoke logging helper implementation.
- `smoke_script_directive.h`: Parsed smoke-script directive/value definitions.
- `smoke_script_runner.h`: Smoke script runner interface.
- `smoke_script_runner.cpp`: Smoke script runner implementation, including tech-tree, phone-panel, and site-protection overlay UI action parsing for scripts.
- `visual_smoke_host_main.cpp`: Visual smoke host executable entry point, including `--port`, optional `--verbose` handling, queue-driven host command submission from HTTP callbacks, the separate site-control lock, tech-tree plus phone-panel and site-protection overlay UI action parsing for browser-driven clicks, the browser-driven harvest and excavate action parser, and a per-frame timing telemetry stream that reports visual-host versus gameplay-DLL cost to the viewer.
